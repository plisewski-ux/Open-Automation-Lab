# 2026-07-02

## Phase

Phase 01 – Identity & Security

## Sprint

Sprint 2 – Administrator Account

## Goal

Replace direct `root` administration with a dedicated administrative user while preserving secure SSH access.

## Decisions

- Created a dedicated administrator account (`piotr`).
- Granted administrative privileges through the `sudo` group.
- Configured SSH public key authentication for the new account.
- Verified passwordless SSH login using the dedicated ED25519 key.
- Decided to keep the `root` account enabled temporarily until several administrative tasks have been successfully completed using the new account.

## Commands Executed

```bash
adduser piotr
usermod -aG sudo piotr

install -d -m 700 -o piotr -g piotr /home/piotr/.ssh
cp /root/.ssh/authorized_keys /home/piotr/.ssh/
chown piotr:piotr /home/piotr/.ssh/authorized_keys
chmod 600 /home/piotr/.ssh/authorized_keys

id piotr
sudo whoami
```

## Verification

Successfully verified:

- administrator account creation,
- membership in the `sudo` group,
- SSH public key authentication,
- passwordless login using the dedicated ED25519 key,
- successful privilege escalation via `sudo`.

Verification output:

```text
piotr@paul154:~$ sudo whoami
root
```

## Next Steps

- Update the local SSH configuration to use the `piotr` account by default.
- Continue all administration tasks using the dedicated administrator account.
- Disable direct `root` login during the SSH Hardening sprint.

## Notes

The migration from `root` to a dedicated administrator account was completed successfully without interrupting administrative access.

The existing `root` account remains available only as a temporary rollback mechanism.
