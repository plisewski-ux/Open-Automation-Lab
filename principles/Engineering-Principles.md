# Foundational Principles

1. Understand before implementing.
2. Document before automating.
3. Automate only what you understand.

# Engineering Principles

## Documentation First

Architecture decisions are documented before implementation.

## Infrastructure as Code

Infrastructure should be reproducible and version controlled.

## Container First

Applications run inside containers whenever possible.

## Human in the Loop

AI assists decision making.
Humans remain responsible for all production changes.

## Security by Default

Security is implemented before exposing services.

## Keep It Simple

Choose the simplest solution that satisfies the requirements.

## Modularity

Every service should have a single responsibility.

## Portability

The platform must remain deployable on different VPS providers.

## Automation

Manual work should eventually become automated.
Automation must remain understandable and maintainable.

## Continuous Improvement

Architecture decisions may be revised when better solutions emerge.
Architecture decisions are expected to evolve as the project matures.

## Observability

Every service should provide sufficient logs, health status and diagnostics to understand its current state.
